# cafe24-skin-archive — 작업 규칙 (반드시 준수)

서지원의 카페24 쇼핑몰 스킨 제작 아카이브. **새 스킨 작업은 모두 이 레포 안에서** 한다.

---

## 1. 폴더 이름 규칙 (자동 적용)

형식: **`<번호>서지원_카페24_<브랜드명>`**

- 번호 = 기존 폴더 중 가장 큰 번호 + 1 (자동 증가)
- 예: `1서지원_카페24_POZE` … `7서지원_카페24_RIPE` → 다음은 `8서지원_카페24_<브랜드>`
- 새 작업 시작은 반드시 다음 명령으로 폴더와 뼈대를 생성한다(번호 자동):

  ```
  python new_skin.py <브랜드명> <포인트색HEX>
  # 예: python new_skin.py LUMINE #c8a24b
  ```

  → 규칙을 모두 만족하는 페이지 뼈대가 새 번호 폴더에 자동 생성된다.

작업이 끝나면 `python gen_readme.py` 로 README의 바로보기 링크를 갱신하고 commit + push 한다.

---

## 2. 페이지 구성 (각각 별도 `.html`, `<head>`에 `<style>` 인라인)

1. **index.html** — 메인: 헤더 + 히어로 + 카테고리 + 추천상품 + 신상품 + 베스트 + 기획전/룩북 띠배너 + 푸터
2. **products.html** — 상품목록: 분류명 + 정렬바 + 상품카드 그리드 + 페이지네이션
3. **product-detail.html** — 상품상세: 큰이미지 + 상품명·가격·옵션·수량·구매버튼 + 상세설명 + (상세/리뷰/문의) 탭
4. **cart.html** — 장바구니: 상품표(체크박스·썸네일·상품명·수량·합계·삭제) + 합계 + 주문버튼
5. **login.html / signup.html** — 로그인 / 회원가입
6. **게시판들** — board.html(목록) / board-view.html(보기) / board-write.html(쓰기)

---

## 3. 핵심 설계 규칙

- **★ 포인트색은 CSS 변수 하나로**: `:root{ --point:#포인트HEX; }`
  → 모든 강조(버튼·가격·배지·링크호버)에 `var(--point)` 사용. 색 하드코딩 금지.
- **★ 메인 각 영역은 독립 `<section>`**: 히어로 / 카테고리 / 추천 / 신상품 / 베스트 / 프로모 각각 `<section>` 하나씩.
  (운영자가 켜고 끄고 순서 변경 가능하도록)
- **★ 이미지 자리 명확히**: 히어로·카테고리썸네일·프로모배너·상품썸네일 전부 `<img>` 로.

---

## 4. ★ 절대 규칙 (안 지키면 카페24에서 깨짐)

1. **상품카드 JS 생성 금지** → HTML에 `<li>` 카드 8개를 직접 복붙(하드코딩)한다.
2. **CSS 외부파일 금지** → 각 HTML `<head>` 안 `<style>` 인라인.
3. **아이콘** = Font Awesome CDN(`<i>`) 또는 인라인 `<svg>`. `<use href>` 스프라이트 금지.
4. **리액트·뷰·번들러 금지**. 순수 HTML + CSS + 최소 바닐라 JS만.
5. **메뉴·카테고리도 HTML 직접** 작성.
6. **헤더·푸터는 모든 페이지 완전 동일**. 서브페이지는 헤더/푸터 사이 중앙 콘텐츠 영역(**max 1550px**) + 페이지 제목.
